******* Interface Temporal - Интерфейс Temporal *******

public interface Temporal extends TemporalAccessor

Интерфейс уровня фреймворка, определяющий доступ для чтения и записи к
временному объекту, такому как дата, время, смещение или некоторая их
комбинация. Это базовый тип интерфейса для объектов даты, времени и смещения,
которые достаточно полны, чтобы ими можно было манипулировать с помощью плюса
и минуса.

Интерфейс реализуется теми классами, которые могут предоставлять и манипулировать
информацией в виде полей или запросов. Большая часть информации о дате и времени
может быть представлена в виде числа. Они моделируются с использованием TemporalField
с числом, хранящимся с использованием long для обработки больших значений.
Год, месяц и день месяца являются простыми примерами полей, но они также включают
мгновенные значения и смещения.
---------------------------------------------------------------------------------
См. ChronoField для стандартного набора полей.
https://docs.oracle.com/javase/8/docs/api/java/time/temporal/ChronoField.html
---------------------------------------------------------------------------------

Две части информации о дате/времени не могут быть представлены числами,
хронологией и часовым поясом. Доступ к ним можно получить с помощью запросов с
использованием статических методов, определенных в TemporalQuery.
---------------------------------------------------------------------------------
https://docs.oracle.com/javase/8/docs/api/java/time/temporal/TemporalQuery.html
---------------------------------------------------------------------------------

Этот интерфейс представляет собой интерфейс уровня фреймворка, который не следует
широко использовать в коде приложения. Вместо этого приложения должны создавать и
передавать экземпляры конкретных типов, таких как LocalDate. Для этого есть много
причин, одна из которых заключается в том, что реализации этого интерфейса могут
быть в календарных системах, отличных от ISO.
---------------------------------------------------------------------------------
См. ChronoLocalDate для более полного обсуждения проблем.
https://docs.oracle.com/javase/8/docs/api/java/time/chrono/ChronoLocalDate.html
---------------------------------------------------------------------------------

Класс может реализовать этот интерфейс, если он соответствует трем критериям:
- он обеспечивает доступ к информации о дате/времени/смещении
  согласно TemporalAccessor;
- набор полей является смежным от самого большого к самому маленькому;
- набор полей завершен, так что никакое другое поле не требуется для определения
  допустимого диапазона значений для представленных полей.

Четыре примера подтверждают это:
- LocalDate реализует этот интерфейс, поскольку он представляет собой набор полей,
  которые являются непрерывными от дней до бесконечности и не требуют никакой внешней
  информации для определения достоверности каждой даты. Следовательно, он может
  правильно реализовать плюс/минус.
- LocalTime реализует этот интерфейс, поскольку он представляет собой набор полей,
  которые являются смежными от нано до нескольких дней и не требуют никакой внешней
  информации для определения достоверности. Он умеет правильно реализовывать плюс/минус,
  оборачивая день.

- MonthDay, комбинация месяца года и дня месяца, не реализует этот интерфейс. В то время
  как комбинация является непрерывной, от дней до месяцев в пределах лет, комбинация не
  имеет достаточной информации, чтобы определить допустимый диапазон значений для дня месяца.
  Таким образом, он не может правильно реализовать плюс/минус.
- Комбинация дня недели и дня месяца ("пятница, 13-е") не должна реализовывать этот интерфейс.
  Он не представляет собой непрерывный набор полей, так как дни и недели перекрываются между
  днями и месяцами.

Требования к реализации: Этот интерфейс не накладывает ограничений на изменяемость реализаций,
однако неизменность настоятельно рекомендуется. Все реализации должны быть сопоставимы.

******* МЕТОДЫ *******

- boolean isSupported(TemporalUnit unit) - Проверяет, поддерживается
  ли указанный модуль.

- default Temporal minus(long amountToSubtract, TemporalUnit unit) -
  Возвращает объект того же типа, что и этот объект, за вычетом
  указанного периода.

- default Temporal minus(TemporalAmount amount) - Возвращает объект
  того же типа, что и этот объект, с вычтенной суммой.

- Temporal plus(long amountToAdd, TemporalUnit unit) - Возвращает объект
  того же типа, что и этот объект, с добавлением указанного периода.

- default Temporal plus(TemporalAmount amount) - Возвращает объект того
  же типа, что и этот объект, с добавленной суммой.

- long until(Temporal endExclusive, TemporalUnit unit) - Вычисляет количество
  времени до другого временного интервала в единицах измерения.

- default Temporal with(TemporalAdjuster adjuster) - Возвращает скорректированный
  объект того же типа, что и этот объект, с внесенными корректировками.

- Temporal with(TemporalField field, long newValue) - Возвращает объект того же
  типа, что и этот объект, но с измененным указанным полем.
